\# 프로젝트 컨텍스트 — 공구반장 (Gonggu Banjang)



목적

\- 스마트폰 카메라로 공구를 촬영 → 이미지 인식 AI로 공구 정보 자동추출

\- 100개 미만의 공구/장비를 대상(가정/학급/소호/동아리)

\- 100% 무료, 태그/QR/NFC/GPS 불사용



디자인/브랜딩

\- TDS 톤 차용: 레드(#FF3040)/블랙/화이트

\- 입체형 사각 버튼, 심플 인터랙션

\- 목록 썸네일 + 자세히보기(플로팅 큰 이미지)



기술스택

\- 프런트: Vite + React + TypeScript (웹뷰)

\- 백엔드: Node.js(Express, TypeScript)

\- AI: Google AI Studio (Gemini 2.5 이미지 이해 계열) — 서버측 호출

\- 배포: 웹뷰(앱인토스), API는 Cloud Run 등

\- 환경: VS Code/Cursor, Node.js, Android Studio+ADB, HTTPS(운영 필수)



핵심 기능(초기)

1\) 촬영/업로드 → AI 추출(JSON 스키마 강제)

2\) 공구 DB화(이름/제조사/모델/분류/스펙/상태/구매일/수명)

3\) 대출/반납 기록(사용자/일시)

4\) 사건(파손/분실/수리) 기록

5\) 신품/중고 구분

6\) 필수/미사용 데이터 사용자 선택(requiredKeys/hiddenKeys)

7\) 목록 썸네일 + 플로팅 큰 사진



정책/보안

\- 서버에 이미지 상주 금지(임시파일 즉시 삭제)

\- 운영 HTTPS 강제, 테스트에서만 HTTP 허용

\- CORS: 프런트/앱인토스 도메인만 허용

\- 외부 로깅 솔루션 제한 준수



품질/제약

\- 번들(압축해제) 100MB 이하

\- iOS 3rd-party 쿠키 차단 고려(쿠키 의존 X)

\- 네트워크 실패/재시도/오프라인 대비 UX
